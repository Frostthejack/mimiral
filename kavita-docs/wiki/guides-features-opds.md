OPDS support is enabled by default on all Kavita installs. This will grant all your users a unique URL and Auth key to access your server from external readers. You can use any Auth Key with OPDS. From this UI, a user can use the copy button to quickly copy it into their clipboard. If the admin has set the hostname of the server, then the OPDS URL will be the full domain name. Kavita supports OPDS and the expanded page streaming specification called OPDS-PS. The expanded OPDS-PS can support images, but not as many clients have support for it. While OPDS and OPDS-PS cover a lot of use cases, the absolute best way to interact with the content on your server is through KavitaтАЩs API. Clients that use the API have a richer experience and can support new advanced features as they are added to Kavita. Kavita Implementation Since Kavita supports rich collections like Smart Filters, Collections, Want to Read, Reading Lists, and Series, the OPDS feed will have different entries for each. Search works across these entities (assuming OpenSearch support) and all feeds have pagination support. Kavita also offers a few enhancements not traditionally found in other OPDS Servers (these are configurable in User Settings): Continue From If you have reading progress on a Series/Volume/Reading List, Kavita will generate a тАЬContinue From XтАЭ item at the top of the first page. This is an alias of the underlying file and allows you to skip scrolling until you find your location. Progress Encoded Icons Kavita also encodes reading progress directly into the title (since metadata support is limited in clients).
